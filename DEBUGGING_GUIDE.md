# 房间选择系统 - 调试指南

## 🔧 如果出现"交互失败"错误

### 步骤1: 检查服务器日志

运行服务器时，仔细查看控制台输出：

```bash
cd TRPGWebDiscord/backend
python server.py
```

**关键日志信息**：
- `📢 [准备]` - 用户执行了 !准备 命令
- `✅ [准备] 显示房间选择菜单` - 菜单已显示
- `🔍 [Modal] 用户输入房间号: XXX` - 用户输入了房间号
- `❌ [Modal] 房间不存在: XXX` - 房间号不存在
- `❌ [Modal Error]` - 出现了错误

### 步骤2: 完整测试流程


### 步骤3: 常见问题诊断

#### 问题：输入房间号后提示"房间号不存在"
**可能原因**：
1. 房间号输入错误了
2. 房间已经被删除（Bot重启或频道删除）
3. 房间号确实没被创建

**解决方案**：
- 使用 `!房间列表` 查看当前存在的所有房间号
- 确保房间创建者还在线且Bot还在运行

#### 问题：点击加入房间按钮后没反应
**可能原因**：
1. Bot权限不足
2. Discord API 超时
3. 模态框初始化失败

**解决方案**：
- 检查Bot是否在线（查看服务器日志）
- 等待几秒钟再试
- 重启Bot

#### 问题：交互失败（立即出现）
**可能原因**：
1. 模态框响应超时（3秒内未响应）
2. Bot没有权限
3. 网络连接问题

**解决方案**：
- 查看服务器日志中的 `❌ [Modal Error]`
- 确保Bot有发送私信的权限
- 检查网络连接

## ✅ 快速验证脚本

复制以下代码到 Python 控制台测试房间系统：

```python
# 测试房间号生成
import random

def test_room_generation():
    rooms = set()
    for i in range(10):
        room_num = f"{random.randint(100000, 999999)}"
        print(f"生成房间号 {i}: {room_num}")
        if room_num in rooms:
            print(f"  ❌ 重复!")
        else:
            print(f"  ✅ 唯一")
        rooms.add(room_num)

test_room_generation()
```
