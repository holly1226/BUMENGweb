# 房间隔离系统 - 测试指南

## 🚀 快速开始

### 1. 启动 Bot

```bash
cd TRPGWebDiscord/backend
python server.py
```

查看日志输出应该看到：
```
✅ Discord Bot 已上线: [Bot Name]
🎤 正在初始化 ChatTTS...
🎤 ChatTTS 初始化完成！
```

### 2. 准备 Discord

- 创建一个主游戏频道（例如 `#main-game`）
- 邀请 Bot 和至少 3 个测试用户加入

## 🧪 完整测试流程

### 测试 1：基础房间创建

**目的**：验证玩家可以创建房间

**步骤**：
1. 玩家 A 在 `#main-game` 输入 `!准备`
2. 看到房间选择菜单，有两个按钮
3. 点击 **「🎮 创建房间」**

**预期结果**：
- 私信收到：`✅ 房间创建成功！🎯 房间号: XXXXXX`
- 记记住房间号（例如 234567）
- 服务器日志输出：
  ```
  🎮 [CreateRoom] 用户 playerA 开始创建房间...
  ✅ [CreateRoom] 已延迟响应
  ✅ [MultiRoom] 创建房间: main-game | 房间号: 234567 | 频道ID: XXX | 创建者: playerA
  👤 [MultiRoom] 玩家 playerA 加入房间 234567
  ✅ [CreateRoom] 房间已创建，房间号: 234567
  ✅ [CreateRoom] 创建房间成功！
  ```

### 测试 2：房间加入

**目的**：验证其他玩家可以加入已有房间

**步骤**：
1. 玩家 B 在 `#main-game` 输入 `!准备`
2. 点击 **「🔓 加入房间」**
3. 出现对话框，输入房间号 `234567`
4. 提交

**预期结果**：
- 私信收到：`✅ 成功加入房间！🎯 房间号: 234567`
- 服务器日志输出：
  ```
  🔍 [Modal] 用户 playerB 开始加入房间...
  🔍 [Modal] 输入的房间号: 234567
  ✅ [Modal] 已延迟响应
  ✅ [Modal] 房间存在: 234567
  👤 [MultiRoom] 玩家 playerB 加入房间 234567
  ✅ [Modal] 成功加入房间！
  ```

### 测试 3：消息隔离（关键测试）

**目的**：验证不同房间的消息互不可见

**流程**：

1. **创建两个房间**
   - 玩家 A 创建房间 `234567` ✅
   - 玩家 B 加入房间 `234567` ✅
   - 玩家 C 创建房间 `345678` ✅
   - 玩家 D 加入房间 `345678` ✅

2. **发送测试消息**
   - 玩家 A 在 `#main-game` 发送：`你好，这是房间A的消息`
   - 玩家 C 在 `#main-game` 发送：`你好，这是房间B的消息`

3. **验证隔离效果**

**预期日志输出**：
```
# 玩家 A 的消息
⚠️ [Message Filter] 玩家 A 的消息处理
👤 房间 234567 收到消息：你好，这是房间A的消息

# 玩家 C 的消息
⚠️ [Message Filter] 玩家 C 的消息处理
👤 房间 345678 收到消息：你好，这是房间B的消息

# 关键：房间 234567 不会收到房间 345678 的消息
```

**验证方式**：
- 查看 `engine.py` 的日志输出
- 房间 234567 应该只收到玩家 A 和 B 的消息
- 房间 345678 应该只收到玩家 C 和 D 的消息

### 测试 4：没加入房间的消息过滤

**目的**：验证未加入房间的玩家消息被忽略

**步骤**：
1. 玩家 E 直接在 `#main-game` 发送消息（不先运行 `!准备`）

**预期结果**：
- 消息被完全忽略
- 服务器日志输出：
  ```
  ⚠️ [Message Filter] 玩家 E 的消息被过滤：还未加入房间
  ```

### 测试 5：命令隔离

**目的**：验证命令只在玩家所在的房间内执行

**步骤**：
1. 玩家 A（在房间 234567）输入 `!建议 龙族背景`
2. 玩家 C（在房间 345678）输入 `!建议 魔法背景`

**预期结果**：
- 两条建议被发送到各自的房间
- 服务器日志：
  ```
  # 玩家 A 的命令
  👤 [MultiRoom] 玩家 A 在房间 234567 中提交建议

  # 玩家 C 的命令
  👤 [MultiRoom] 玩家 C 在房间 345678 中提交建议
  ```
- 房间 A 的成员能看到"龙族背景"建议
- 房间 B 的成员能看到"魔法背景"建议
- 两个房间看不到对方的建议

### 测试 6：!房间列表

**目的**：验证房间列表显示所有房间

**步骤**：
1. 创建了房间 234567 和 345678 后
2. 任意玩家输入 `!房间列表`

**预期结果**：
- 显示两个房间的信息
- 包括房间号、玩家数、状态等
- 玩家可以看到另一个房间的存在（但看不到其消息）

### 测试 7：游戏开始

**目的**：验证游戏只在特定房间内开始

**步骤**：
1. 房间 A（234567）中，玩家 A 输入 `!开始游戏`
2. 房间 B（345678）中，玩家 C 输入 `!开始游戏`

**预期结果**：
- 两个房间分别开始游戏
- 房间 A 的消息只影响房间 A
- 房间 B 的消息只影响房间 B
- 互不干扰

## 🔍 日志过滤关键字

在查看日志时，使用这些关键字快速定位问题：

- `[Message Filter]` - 消息过滤相关
- `[MultiRoom]` - 房间操作
- `👤` - 玩家加入/离开房间
- `📢 [准备]` - 准备命令
- `🎮 [CreateRoom]` - 创建房间
- `🔍 [Modal]` - 加入房间模态框
- `❌ [Message Filter]` - 消息被过滤（玩家未加入房间）

## 🐛 常见问题排查

### 问题 1：玩家的消息没有出现

**可能原因**：
- 玩家还未加入任何房间
- 房间号不存在或已被删除

**排查步骤**：
1. 检查日志是否有 `⚠️ [Message Filter] 玩家 XXX 的消息被过滤`
2. 玩家应先运行 `!准备` 加入或创建房间

### 问题 2：不同房间的消息混在一起

**可能原因**：
- on_message 过滤逻辑有问题
- 玩家名称冲突

**排查步骤**：
1. 检查 server.py 中的 on_message 实现
2. 验证 user_to_room 映射是否正确
3. 检查玩家名称是否唯一

### 问题 3：命令只在一个房间执行

**可能原因**：
- 这是正确的行为！命令只在玩家所在的房间内执行
- 如果两个房间都执行了命令，应该看到房间号不同的日志输出
