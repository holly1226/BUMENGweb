## 第四步: 正常使用

系统已自动集成到 `manager.py`，无需额外操作!

当调用 `start_game()` 时，系统会自动:
1. 提取玩家建议和角色偏好
2. 检索相似的文学作品
3. 匹配适合的高光时刻
4. 将结果注入AI的剧情推演prompt

## 常见问题

### Q: 忘记运行初始化怎么办?
A: 第一次调用时会自动构建索引，但会比较慢。建议提前运行初始化脚本。

### Q: 如何查看传递给AI的完整prompt?
A: 运行 `python debug_prompt.py` 或查看游戏运行时的控制台输出

### Q: 如何更新文学作品库?
A: 修改 `literature_mentor.json` 后，重新运行:
```bash
rm -rf storage/embeddings_cache/
python init_search_engine.py
```

### Q: 索引文件在哪里?
A: `backend/storage/embeddings_cache/`

### Q: 可以删除缓存吗?
A: 可以,删除后会自动重建,但需要5-10分钟。

### Q: Debug输出太多,如何关闭?
A: 在 `manager.py` 中注释掉 `_generate_deep_plot_inspection` 里的 `print` 语句

## 🔧 调试技巧

### 1. 测试单个检索
```bash
python debug_prompt.py
# 选择选项 2
```

### 2. 保存prompt到文件
在 `debug_prompt.py` 运行结束时选择 `y` 保存到 `debug_prompt_output.txt`

### 3. 查看检索质量
观察相似度分数 (similarity_score):
- `> 0.8`: 非常相似
- `0.6-0.8`: 比较相似
- `< 0.6`: 相似度较低

### 4. 调整检索参数
在 `_generate_deep_plot_inspection` 中修改:
```python
search_results = await search_engine.search_for_game_init(
    suggestions=suggestions,
    role_prefs=role_prefs,
    literature_top_k=5,      # 调整文学作品数量
    highpoint_top_k=3        # 调整高光时刻数量
)
```
