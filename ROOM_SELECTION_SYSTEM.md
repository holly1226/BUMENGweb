# TRPGWebDiscord 房间选择系统

## 🆕 新功能概述

现在 TRPGWebDiscord 支持**数字房间号系统**！玩家可以：
- 🎮 **创建房间** - 自动生成一个6位房间号
- 🔓 **加入房间** - 输入房间号加入现有房间
- 📋 **查看房间列表** - 查看所有活跃房间

## 🎯 使用流程

### 第一步：输入 `!准备` 命令

```
用户输入: !准备
↓
机器人显示菜单：
  [🎮 创建房间]  [🔓 加入房间]
```

### 第二步：选择操作

#### 选项 A：创建房间
```
点击 [🎮 创建房间] 按钮
↓
✅ 房间创建成功！
🎯 房间号: 123456
📍 频道: #game-room-1
↓
机器人在该频道启动大厅，显示：
  📢 灵感与角色征集开启！
  📊 当前识别到玩家 (3人)...
  1️⃣ 世界观建议：输入 !建议 [内容]
  2️⃣ 个人要求：输入 !角色 [性别/性格]
  💡 所有人准备好后，输入 !开始游戏 正式开局！
```

#### 选项 B：加入房间
```
点击 [🔓 加入房间] 按钮
↓
弹出对话框：输入房间号
↓
用户输入：123456
↓
系统检查房间是否存在...
  ✅ 房间存在 → 进入该房间
  ❌ 房间不存在 → 错误提示
```

### 第三步：加入房间后

玩家加入房间后，会在该房间的频道中看到游戏信息并可以开始游戏。

## 🎮 房间号详解

### 房间号格式
- **6位数字** - 例如 `123456`
- **全局唯一** - 每个房间有唯一的房间号
- **自动生成** - 创建房间时自动分配

### 房间号的用途
```
创建房间时：
  用户1 在 #game-room-1 创建房间 → 房间号: 234567

加入房间时：
  用户2 输入房间号: 234567
  系统自动跳转到 #game-room-1 加入该房间
```

## 📊 新增命令

### `!准备`
弹出房间选择菜单，让用户选择创建或加入房间。

**使用场景**：
- 第一次进入游戏
- 想加入其他房间

### `!房间列表`
显示所有活跃房间的列表（包括房间号、状态、频道、玩家数等）。

**输出例子**：
```
🏢 所有活跃房间

🎯 房间号: `234567`
状态: 🟢 进行中
玩家: 3
频道: #game-room-1
场景: 黑暗森林

🎯 房间号: `345678`
状态: 🔴 等待中
玩家: 2
频道: #game-room-2
场景: 酒馆大厅
```

## 📝 实现细节

### 房间号生成
```python
# 每次创建房间都会生成一个不重复的6位数字
room_number = f"{random.randint(100000, 999999)}"

# 检查是否已存在
while room_number in existing_rooms:
    room_number = f"{random.randint(100000, 999999)}"
```

### 房间存储
```python
# 两种查找方式
multi_room_engine.rooms_by_number   # {房间号: GameRoom}
multi_room_engine.rooms_by_channel  # {频道ID: GameRoom}

room = multi_room_engine.get_room_by_number("234567")
room = multi_room_engine.get_room_by_channel(channel_id)
```

## 📌 常见问题

### Q: 房间号是什么？
A: 房间号是一个6位的数字，用来唯一标识游戏房间。不同房间有不同的房间号。

### Q: 我怎么知道房间号？
A:
- 创建房间时，机器人会告诉您
- 向房间创建者询问
- 使用 `!房间列表` 查看所有房间号

### Q: 房间号会重复吗？
A: 不会。系统确保每个房间号都是唯一的。

### Q: 我可以自定义房间号吗？
A: 不可以。房间号由系统自动生成。

### Q: 房间会一直存在吗？
A: 房间在频道或Bot重启后会被删除。如果需要持久化，后续可以添加数据库支存功能。

## 🔧 故障排查

### 问题：输入房间号后说"房间号不存在"
**解决方案**：
- 检查房间号是否输入正确（6位数字）
- 使用 `!房间列表` 查看实际的房间号
- 确保房间创建者的Bot还在运行

### 问题：能进入房间但看不到游戏信息
**解决方案**：
- 等一下，机器人可能正在加载
- 刷新Discord界面
- 联系房间创建者重新启动游戏

### 问题：频道显示"已经有房间了"
**解决方案**：
- 这个频道已经创建过房间
- 如果想创建新房间，请在另一个频道执行 `!准备`
- 或者执行 `!重置` 来清除当前房间状态
